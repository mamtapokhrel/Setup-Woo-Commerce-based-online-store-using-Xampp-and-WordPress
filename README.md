Overview
Trend Setter is a functional online clothing store built from the ground up on a self-configured local server. Where a hosted SaaS platform hides the infrastructure, this project exposes it: the server stack, the database, and the CMS were each installed and wired together manually before a single product was added.
The store sells women's fashion    tops, dresses, and co-ord sets  with full product listings, a working cart, and an end-to-end checkout.

What I built
•	Server environment: installed and configured XAMPP; ran Apache and MySQL locally to serve the site
•	Database: created and connected a MySQL database (trendsetterdb) via phpMyAdmin, then linked it to WordPress during installation
•	CMS + theme: installed WordPress in htdocs, then installed and activated the WooCommerce-ready Storefront theme
•	Store setup: ran the WooCommerce setup wizard to configure store details, currency (NPR), payment methods, and shipping
•	Information architecture: built a product taxonomy categories (Tops, Dresses, Co-ord Sets), tags, brands, and SEO-friendly slugs
•	Product catalogue: published listings with structured descriptions, specifications, pricing, sale prices, and product imagery
•	Transaction flow: tested the full customer journey product page → add to cart → cart totals → checkout → billing details → Cash on Delivery → order confirmation

Key takeaways
Understanding what sits underneath a WordPress site the server, the database, the URL structure, the taxonomy is what makes it possible to diagnose and optimize one later, rather than only decorate it.
